**功能分支**: `009-sdd-summary-slides`
**建立日期**: 2026-02-18
**狀態**: 草案
**輸入**: 使用者描述: "請根據此專案介紹 SDD 所有的內容，產生一組投影片總結所有的主題"

## 使用者場景與測試 *(強制項目)*

### 使用者故事 1 - 總覽 SDD 核心價值 (優先級: P1)

作為觀眾，我希望在最後能快速回顧 SDD 的核心精神，以加深印象。

**為何以此優先級**: 這是整個簡報的收尾，重點在於強化「規格即代碼」的核心訊息。

**獨立測試**: 確認投影片包含「規格即真理」、「自動化驗證」與「持續演進」的三大支柱總結。

**驗收場景**:

1. **假設** 投影片載入, **當** 翻至總結章節, **則** 應展示 SDD 核心價值的金字塔圖或摘要列表。

---

### 使用者故事 2 - 回顧工具與流程 (優先級: P1)

作為開發者，我希望有一張全景圖能串聯所有指令 (`/speckit.*`) 與開發階段。

**為何以此優先級**: 幫助使用者將分散的指令知識點串聯成完整的 workflow。

**獨立測試**: 確認投影片包含一張整合了 Specify, Plan, Implement, Checklist, Clarify 的完整流程圖。

**驗收場景**:

1. **假設** 正在閱讀總結, **當** 查看流程總覽頁面, **則** 應能清楚看到從 `spec.md` 到代碼產出的完整路徑。

---

## 需求 *(強制項目)*

### 功能需求

- **FR-001**: 必須新增一個獨立的 Markdown 檔案 `slides/08-summary.md`。
- **FR-002**: 內容必須回顧以下主題：
  1. **SDD 核心**: 規格驅動、測試優先、文件即代碼。
  2. **工具鏈**: Spec Kit 指令集 (`speckit.specify`, `plan`, `implement`)。
  3. **品質機制**: 不確定性處理 (No Guessing) 與除錯流程 (Fix Recipe)。
- **FR-003**: 必須包含「下一步行動 (Call to Action)」，引導使用者開始使用 Spec Kit。
- **FR-004**: 必須在 `index.html` 中載入此新章節，置於除錯流程之後。
- **FR-005**: 必須使用 Mermaid 心智圖 (Mindmap) 或流程圖總結 SDD 知識體系。

### 憲章合規性 (Constitution Compliance)

- **CC-001**: 投影片內容「必須」儲存在外部 `.md` 檔案中。
- **CC-002**: 技術圖表「必須」使用 Mermaid.js 語法。
- **CC-003**: 佈局「必須」針對 21:9 (1920x1080) 進行優化。
- **CC-004**: 部署「必須」透過 Actions 自動化至 GitHub Pages。
- **CC-005**: 內容與文件「必須」使用正體中文。
