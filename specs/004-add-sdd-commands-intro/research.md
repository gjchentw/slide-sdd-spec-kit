# 研究報告：新增 Spec Kit 指令與 SDD 流程對照投影片

## 決策 1：指令與階段映射

- **問題**: 如何清晰呈現指令與開發階段的對應關係？
- **決策**: 採用「階段 - 指令 - 作用」的三層結構進行說明，並將 6 個階段拆分為多個垂直投影片 (Vertical Slides) 以避免單頁資訊過載。
- **理由**: 每個階段都有豐富的描述（時機、重點），單頁無法容納。

## 決策 2：Mermaid 流程圖優化

- **問題**: 原文提供的 Mermaid `graph TD` (Top-Down) 可能在 21:9 螢幕上過於細長。
- **決策**: 嘗試使用 `graph LR` (Left-Right) 或保留 `TD` 但透過 CSS 縮放。
- **修正**: 考慮到指令流程的線性特性，`graph LR` 可能更適合寬螢幕。但為了忠實還原使用者輸入的結構，將優先使用 `graph TD` 並觀察效果，必要時調整。
- **更新**: 經評估，若節點過多，`graph LR` 更佳。決定將圖表調整為 **`graph LR`** 以適配 1920px 寬度。

## 決策 3：檔案整合

- **問題**: 新投影片的位置？
- **決策**: 建立 `slides/03-sdd-commands.md`，置於 Spec Kit 核心概念之後。
- **理由**: 邏輯順序：原理 -> 核心概念 -> 實際操作指令。
