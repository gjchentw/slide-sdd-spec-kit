# 功能規格：新增 SDD 運作原理介紹投影片

**功能分支**: `002-add-sdd-intro`  
**建立日期**: 2026-02-18  
**狀態**: 草案  
**輸入**: 使用者描述: "請新增介紹 SDD 的主題，請讀取 https://raw.githubusercontent.com/github/spec-kit/refs/heads/main/spec-driven.md 新增一組投影片介紹 SDD 運作的原理，並附上 mermaid 流程圖"

## 使用者場景與測試 *(強制項目)*

### 使用者故事 1 - 瀏覽 SDD 運作原理 (優先級: P1)

作為聽眾，我希望在投影片中看到關於「規格驅動開發 (SDD)」運作原理的詳細介紹，以便理解此開發模式的核心概念。

**為何以此優先級**: 這是本功能的核心需求，旨在傳遞知識。

**獨立測試**: 打開投影片，導航至 SDD 介紹章節，確認內容包含 SDD 核心概念與流程圖。

**驗收場景**:

1. **假設** 投影片已載入，**當** 使用者翻頁至 SDD 介紹章節，**則** 應顯示標題為「規格驅動開發 (SDD) 原理」的頁面，並包含重點文字與 Mermaid 流程圖。

---

## 需求 *(強制項目)*

### 功能需求

- **FR-001**: 必須新增一個獨立的 Markdown 檔案（如 `slides/01-sdd-intro.md`）來存放 SDD 介紹內容。
- **FR-002**: 內容必須基於 `spec-driven.md` 的核心概念，至少包含「權力反轉 (The Power Inversion)」與「SDD 工作流 (The SDD Workflow)」的重點摘要。
- **FR-003**: 必須包含一張 Mermaid 流程圖，視覺化展示 SDD 的運作流程（例如：Idea -> Spec -> Plan -> Code）。
- **FR-004**: 投影片佈局必須符合 21:9 寬螢幕設計規範。
- **FR-005**: 所有說明文字必須使用正體中文。

### 憲章合規性 (Constitution Compliance)

- **CC-001**: 投影片內容「必須」儲存在外部 `.md` 檔案中。
- **CC-002**: 技術圖表「必須」使用 Mermaid.js 語法。
- **CC-003**: 佈局「必須」針對 21:9 (1920x1080) 進行優化。
- **CC-004**: 部署「必須」透過 Actions 自動化至 GitHub Pages。
- **CC-005**: 內容與文件「必須」使用正體中文。

## 成功準則 *(強制項目)*

### 可衡量結果

- **SC-001**: SDD 介紹章節包含至少 3 個關鍵概念的摘要。
- **SC-002**: Mermaid 流程圖在投影片中正確渲染，無語法錯誤。
- **SC-003**: 內容文字經人工檢核為流暢的正體中文。

### 邊界案例

- **內容過長**: 若摘要文字過多導致單頁無法顯示，應拆分為垂直投影片 (Vertical Slides) 或多頁。
- **圖表過大**: 若 Mermaid 圖表過寬或過高，應使用 CSS 縮放或調整佈局以適應 1920x1080 空間。
