# 功能規格：新增 Spec Kit 核心概念介紹投影片

**功能分支**: `003-add-spec-kit-intro`  
**建立日期**: 2026-02-18  
**狀態**: 草案  
**輸入**: 使用者描述: "請新增介紹 spec-kit 的主題，請讀取 https://github.github.com/spec-kit/ 產生一組投影片介紹 spec-kit 的核心概念"

## 使用者場景與測試 *(強制項目)*

### 使用者故事 1 - 了解 Spec Kit 核心哲學 (優先級: P1)

作為聽眾，我希望在投影片中看到 Spec Kit 的「核心哲學 (Core Philosophy)」，以便理解其為何能加速高品質軟體的構建。

**為何以此優先級**: 這是 Spec Kit 的價值主張，必須首先傳遞。

**獨立測試**: 導航至 Spec Kit 介紹章節，確認頁面列出「意圖驅動」、「豐富規格」、「多步精煉」等核心概念。

**驗收場景**:

1. **假設** 投影片已載入，**當** 使用者翻頁至 Spec Kit 介紹，**則** 應顯示標題為「Spec Kit 核心哲學」的頁面，並包含來自官網的關鍵點。

### 使用者故事 2 - 了解開發階段 (優先級: P2)

作為開發者，我希望了解 Spec Kit 支援的「開發階段 (Development Phases)」，以便知道它適用於哪些場景（如從零開始、創意探索、迭代增強）。

**為何以此優先級**: 讓聽眾了解工具的適用範圍。

**獨立測試**: 確認投影片中有一頁專門介紹 Greenfield, Creative Exploration, Brownfield 三種階段。

---

## 需求 *(強制項目)*

### 功能需求

- **FR-001**: 必須新增一個獨立的 Markdown 檔案（如 `slides/02-spec-kit-intro.md`）。
- **FR-002**: 內容必須基於 `https://github.github.com/spec-kit/` 的 "Core Philosophy" 章節，翻譯並整理為正體中文重點。
- **FR-003**: 內容必須包含 "Development Phases" 的表格或清單，涵蓋 0-to-1, Exploration, Iterative 三種模式。
- **FR-004**: 投影片佈局必須符合 21:9 寬螢幕設計規範。
- **FR-005**: 必須在 `index.html` 中載入此新章節，置於 SDD 原理之後。

### 憲章合規性 (Constitution Compliance)

- **CC-001**: 投影片內容「必須」儲存在外部 `.md` 檔案中。
- **CC-002**: 若有圖表，「必須」使用 Mermaid.js (本功能主要為文字與表格)。
- **CC-003**: 佈局「必須」針對 21:9 (1920x1080) 進行優化。
- **CC-004**: 部署「必須」透過 Actions 自動化至 GitHub Pages。
- **CC-005**: 內容與文件「必須」使用正體中文。

## 成功準則 *(強制項目)*

### 可衡量結果

- **SC-001**: 投影片包含「核心哲學」與「開發階段」兩個主要部分。
- **SC-002**: 所有英文術語（如 Greenfield, Brownfield）旁皆有適當的正體中文解釋。
- **SC-003**: 內容排版在 1920x1080 解析度下無溢出。

### 邊界案例

- **術語翻譯**: 若某些術語（如 Vibe-coding）無標準譯名，應保留原文並加註解釋，避免誤導。
- **表格過長**: 若「開發階段」表格內容過多，應適當縮減或拆分頁面，確保可讀性。
