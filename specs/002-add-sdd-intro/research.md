# 研究報告：新增 SDD 運作原理介紹投影片

## 決策 1：SDD 流程圖語法

- **問題**: 如何視覺化 SDD 流程？
- **決策**: 使用 Mermaid.js 的 `graph LR` (左至右) 語法。
- **理由**: 簡單直觀，適合呈現線性流程 (Idea -> Spec -> Plan -> Code)。
- **替代方案**: `sequenceDiagram` (太複雜)，`flowchart TD` (垂直佔用太多空間，不利於 21:9 佈局)。

## 決策 2：獨立檔案結構

- **問題**: 如何組織新增的投影片內容？
- **決策**: 建立 `slides/01-sdd-intro.md`，並在 `index.html` 中新增一個 `<section>` 區塊來載入。
- **理由**: 保持內容模組化，避免單一大檔難以維護。

## 決策 3：內容摘要策略

- **問題**: 原文 `spec-driven.md` 內容豐富，如何濃縮至投影片？
- **決策**: 提取「權力反轉」作為核心概念頁，並將「工作流」簡化為步驟列表配合圖表。
- **理由**: 投影片應精簡重點，細節留待口頭說明。
