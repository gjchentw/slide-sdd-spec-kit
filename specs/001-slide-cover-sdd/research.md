# 研究報告：投影片封面與 SDD 開發主題

## 決策 1：Git Config 資訊注入

- **問題**: GitHub Pages 為靜態網站，無法在執行時讀取伺服器端的 `git config`。
- **決策**: 在建置階段 (Build Time) 透過腳本讀取 `git config user.name` 與 `user.email`，並將其寫入 `slides/config.json` 或直接替換模板中的佔位符。
- **理由**: 確保資訊來源為建置當下的環境配置，符合需求「動態讀取」。
- **替代方案**: 使用 URL 參數傳遞 (URL query params)，但使用者體驗不佳。

## 決策 2：外部 Markdown 載入

- **問題**: 如何高效管理投影片內容？
- **決策**: 使用 reveal.js 的 `data-markdown` 屬性，將每張投影片（或章節）拆分為獨立的 `.md` 檔案。
- **理由**: 符合憲章「外部 Markdown 內容」原則，利於版本控制。

## 決策 3：解析度與長寬比

- **問題**: 憲章規定「21:9 (1920x1080)」。但 1920x1080 實為 16:9。
- **決策**: 依據憲章明確指定的像素值 **1920x1080** 設定 reveal.js 的 `width` 與 `height`。若需達成 21:9 視覺效果，可透過 CSS 調整內容邊距，或後續修正憲章定義。
- **理由**: 1920x1080 為最通用的高畫質標準，確保大多數設備的相容性。
