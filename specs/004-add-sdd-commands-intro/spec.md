# 功能規格：新增 Spec Kit 指令與 SDD 流程對照投影片

**功能分支**: `004-add-sdd-commands-intro`
**建立日期**: 2026-02-18
**狀態**: 草案
**輸入**: 使用者描述：新增介紹 spec-kit 指令對應 SDD 流程的投影片，包含 6 個階段（憲章、啟動、提煉、計畫、拆解、執行）與 Mermaid 流程圖。

## 使用者場景與測試 *(強制項目)*

### 使用者故事 1 - 了解 SDD 指令流程 (優先級: P1)

作為開發者，我希望在投影片中看到完整的 Spec Kit 指令操作順序，以便了解如何透過指令實踐 SDD 開發流程。

**為何以此優先級**: 這是 Spec Kit 的操作指南，讓使用者知道如何落地。

**獨立測試**: 導航至指令介紹章節，確認包含 6 個階段的指令說明與 Mermaid 流程圖。

**驗收場景**:

1. **假設** 投影片已載入，**當** 使用者翻頁至指令介紹，**則** 應顯示「Spec Kit 指令流程」章節，並依序介紹 constitution, specify, clarify, plan, tasks, implement 等指令。

---

## 需求 *(強制項目)*

### 功能需求

- **FR-001**: 必須新增一個獨立的 Markdown 檔案（如 `slides/03-sdd-commands.md`）。
- **FR-002**: 內容必須包含 6 個階段的詳細說明：
  1. 確立憲章 (/speckit.constitution)
  2. 啟動新功能 (/speckit.specify)
  3. 提煉與驗收規格 (/speckit.clarify, /speckit.checklist)
  4. 生成實作計畫 (/speckit.plan)
  5. 任務拆解 (/speckit.tasks)
  6. 驗證與實作 (/speckit.analyze, /speckit.implement)
- **FR-003**: 必須包含 Mermaid 流程圖，視覺化展示指令流向（如輸入描述中的 graph TD）。
- **FR-004**: 投影片佈局必須符合 21:9 寬螢幕設計規範。
- **FR-005**: 必須在 `index.html` 中載入此新章節，置於 Spec Kit 核心概念之後。

### 憲章合規性 (Constitution Compliance)

- **CC-001**: 投影片內容「必須」儲存在外部 `.md` 檔案中。
- **CC-002**: 技術圖表「必須」使用 Mermaid.js 語法。
- **CC-003**: 佈局「必須」針對 21:9 (1920x1080) 進行優化。
- **CC-004**: 部署「必須」透過 Actions 自動化至 GitHub Pages。
- **CC-005**: 內容與文件「必須」使用正體中文。

## 成功準則 *(強制項目)*

### 可衡量結果

- **SC-001**: 投影片完整涵蓋 6 個開發階段與對應指令。
- **SC-002**: Mermaid 流程圖正確渲染，且路徑邏輯與描述一致。
- **SC-003**: 內容文字清晰易讀，無排版溢出。

### 邊界案例

- **內容過多**: 6 個階段若在單頁顯示過於擁擠，應拆分為多張垂直投影片。
- **圖表複雜度**: 流程圖若過長，應調整為水平佈局或簡化節點文字。
