# 研究報告：新增 Spec Kit 規格迭代與分支策略投影片

## 決策 1：分支策略視覺化

- **問題**: 如何解釋「分支即膠囊」的概念？
- **決策**: 使用 Mermaid `graph TD` 繪製分支結構，包含 `specs/` 與 `src/` 並存的意象。
- **理由**: 視覺化能更直觀地表達「規格與代碼對齊」的核心價值。

## 決策 2：不確定性處理的呈現

- **問題**: 四種處理機制文字量較大，如何排版？
- **決策**: 將「拒絕猜測」與「品質攔截」分組，「探索性生成」與「持續演進」分組，拆分為兩張投影片。
- **理由**: 避免資訊過載，確保每一點都有足夠的解釋空間。

## 決策 3：檔案整合

- **問題**: 新投影片的位置？
- **決策**: 建立 `slides/04-sdd-iteration.md`，置於 SDD 指令流程之後。
- **理由**: 邏輯順序：指令操作 -> 背後的迭代機制與哲學。
