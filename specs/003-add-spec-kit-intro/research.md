# 研究報告：新增 Spec Kit 核心概念介紹投影片

## 決策 1：內容摘要與翻譯

- **問題**: 如何將英文技術文件轉化為易懂的正體中文投影片？
- **決策**: 
  - **核心哲學**: 翻譯為「意圖驅動開發」、「豐富規格創建」、「多步精煉」、「AI 依賴」。
  - **開發階段**: 翻譯為「從零到一 (Greenfield)」、「創意探索」、「迭代增強 (Brownfield)」。保留英文術語以供對照。
- **理由**: 確保術語準確性同時降低閱讀門檻。

## 決策 2：投影片結構

- **問題**: 內容包含列表與表格，如何呈現？
- **決策**: 
  - 「核心哲學」使用 bullet points。
  - 「開發階段」使用 Markdown 表格，並確保欄寬適中。
- **理由**: 清晰呈現結構化資訊。

## 決策 3：檔案整合

- **問題**: 新投影片的位置？
- **決策**: 建立 `slides/02-spec-kit-intro.md`，並在 `index.html` 中 `01-sdd-intro.md` 之後載入。
- **理由**: 邏輯順序：先懂 SDD 原理，再懂 Spec Kit 工具。
