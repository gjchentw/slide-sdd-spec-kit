# 功能規格：投影片封面與 SDD 開發主題

**功能分支**: `001-slide-cover-sdd`  
**建立日期**: 2026-02-18  
**狀態**: 草案  
**輸入**: 使用者描述: "請建立投影片封面，主題是用 spec-kit 進行 SDD 開發。作者和 email 請用 git config 的資訊"

## 使用者場景與測試 *(強制項目)*

### 使用者故事 1 - 顯示專案封面 (優先級: P1)

當使用者打開投影片時，第一頁必須顯示專案的主題封面，並包含正確的講者資訊。

**為何以此優先級**: 這是投影片的入口，確立主題與講者身份，為聽眾建立第一印象。

**獨立測試**: 打開投影片首頁，確認標題為「使用 spec-kit 進行 SDD 開發」，且作者名稱與 Email 與 git config 設定一致。

**驗收場景**:

1. **假設** git config user.name 為 "TestUser" 且 user.email 為 "test@example.com"，**當** 投影片載入首頁，**則** 標題顯示「使用 spec-kit 進行 SDD 開發」，副標題或作者欄顯示 "TestUser" 及 "test@example.com"。

---

## 需求 *(強制項目)*

### 功能需求

- **FR-001**: 投影片首頁「必須」顯示標題「使用 spec-kit 進行 SDD 開發」。
- **FR-002**: 投影片首頁「必須」動態讀取並顯示 git config 中的 `user.name` 作為作者。
- **FR-003**: 投影片首頁「必須」動態讀取並顯示 git config 中的 `user.email` 作為聯絡資訊。
- **FR-004**: 封面佈局「必須」符合 21:9 寬螢幕設計規範。

### 憲章合規性 (Constitution Compliance)

- **CC-001**: 投影片內容「必須」儲存在外部 `.md` 檔案中（例如 `slides/00-cover.md`）。
- **CC-002**: 技術圖表「必須」使用 Mermaid.js 語法（本功能暫不涉及圖表，但需保留此規範）。
- **CC-003**: 佈局「必須」針對 21:9 (1920x1080) 進行優化。
- **CC-004**: 部署「必須」透過 Actions 自動化至 GitHub Pages。
- **CC-005**: 內容與文件「必須」使用正體中文。

## 成功準則 *(強制項目)*

### 可衡量結果

- **SC-001**: 投影片首頁在 1920x1080 解析度下，內容無跑版或截斷。
- **SC-002**: 變更本地 git config 後重新建置，首頁作者資訊隨之更新。
- **SC-003**: 透過 GitHub Pages 瀏覽時，首頁載入時間小於 1 秒。

### 邊界案例

- 若本地 git config 未設定 user.name 或 user.email，顯示什麼？
  - 預設顯示 "Unknown Author" / "No Email Provided" 或提示錯誤。
- 若 user.name 包含特殊字元或超長字串，是否會破壞版面？
  - 應截斷或自動縮小字體以適應 21:9 版面。
